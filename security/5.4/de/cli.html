Befehlszeile

Die SUSE® Security Kommandozeilenschnittstelle verwenden

Die SUSE® Security Lösung unterstützt eine begrenzte Anzahl von Funktionen über die Kommandozeilenschnittstelle. Die Kommandozeilenschnittstelle wird über den Manager unterstützt, der seinerseits eine RestAPI verwendet, um Befehle an den Controller zu senden. Der Controller verwaltet dann die Enforcer entsprechend. Ein vollständiger Satz von Operationen wird über die REST API unterstützt, die direkt vom Controller bereitgestellt werden kann. Sie können auf die SUSE® Security Kommandozeilenschnittstelle zugreifen, indem Sie den CLI-Befehl für den Manager oder Allinone eingeben, zum Beispiel:

kubectl exec -it neuvector-manager-pod-5bb76b6754-rlmnp -n neuvector -- cli
docker exec -it allinone cli

Dabei ist ‘allinone’ der Container für den Controller. Möglicherweise müssen Sie anstelle des Namens die Container-ID verwenden.

Obwohl die Kommandozeilenschnittstelle über den Manager verfügbar ist, empfehlen wir, die REST API direkt im Controller für Abfragen und Automatisierung zu verwenden.

Beispiele für Befehle der Kommandozeilenschnittstelle

Hier sind einige der häufigsten Befehle der Kommandozeilenschnittstelle:

> login
> logout

Verwenden Sie denselben Benutzernamen/Passwort, den Sie für die Konsole verwenden.

> show policy -h
Usage: cli show policy [OPTIONS] COMMAND [ARGS]...
> create policy rule -h
Usage: cli create policy rule [OPTIONS] FROM TO
> delete policy rule -h
Usage: cli delete policy rule [OPTIONS] ID
> show log -h
Usage: cli show log [OPTIONS] COMMAND [ARGS]...
> set system policy_mode -h
Usage: cli set system policy_mode [OPTIONS] MODE

  Set system policy mode.

Options:
  -h, --help  Show this message and exit.

MODES:
  learn=discover
  evaluate=monitor
  enforce=protect
> set controller <leader_controller_id> debug -c cpath
  Turn on debug mode.
> set controller <leader_controller_id> debug
  Turn off debug mdoe.

Weitere Befehle der Kommandozeilenschnittstelle sind unten aufgeführt.

Packet-Sniffer

Sniffer-Dateien werden im Verzeichnis /var/neuvector/pcap im Enforcer-Container gespeichert. Stellen Sie sicher, dass Sie das Volume mit dem Verzeichnis Ihrer Gastmaschine oder dem lokalen Systemverzeichnis verknüpfen, um auf die Dateien zugreifen zu können. Fügen Sie beispielsweise in der docker-compose-Datei ‘- /var/neuvector:/var/neuvector’ in volumes hinzu.

Um die Paketaufzeichnung auf einem Pod zu starten, müssen Sie die Container-ID kennen, die Sie in das Feld ID_OR_NAME eingeben. Sie können dies mit show container -c <container_name> tun. Starten Sie dann den Sniffer mit request sniffer start <container_id>. Beispiel:
