Inicio de sesión personalizado, encabezado y pie de página

Personalizando el componente de la consola (UI)

Esta guía te ayudará a personalizar la consola con las siguientes opciones:

1. Personaliza el logo de inicio de sesión

Para personalizar el logo de inicio de sesión, sigue estos pasos:

  • Crea un archivo SVG de 300x80 píxeles.

  • Codifica en Base64 el archivo SVG.

  • Guarda el archivo codificado en la variable de entorno CUSTOM_LOGIN_LOGO.

2. Personaliza la directiva de aceptación (por ejemplo, Términos de uso)

Para personalizar la directiva, sigue estos pasos:

  • El contenido de la directiva puede ser HTML o texto plano.

  • Codifica en Base64 el contenido de la directiva.

  • Guarda el contenido codificado en la variable de entorno CUSTOM_EULA_POLICY.

3. Personaliza el banner de la página (se muestra en cada página)

Para personalizar el banner de la página, sigue estos pasos:

Personalización del encabezado

  • Personaliza el contenido y el color del encabezado.

  • El color del banner del encabezado es obligatorio.

  • El valor del color puede ser una palabra clave de color (por ejemplo, amarillo) o un valor Hex (por ejemplo, #ffff00).

  • El contenido es opcional y puede ser una línea de HTML o texto plano con un máximo de 120 caracteres.

  • Codifica en Base64 el contenido del encabezado.

  • Guarda el contenido codificado en las variables de entorno CUSTOM_PAGE_HEADER_COLOR y CUSTOM_PAGE_HEADER_CONTENT.

Personalización del pie de página

  • Personaliza el contenido y el color del pie de página.

  • El color del banner del pie de página será el mismo que el del banner del encabezado si el color no se personaliza.

  • El contenido es opcional y puede ser una línea de HTML o texto plano con un máximo de 120 caracteres.

  • Codifica en Base64 el contenido del pie de página.

  • Guarda el contenido codificado en las variables de entorno CUSTOM_PAGE_FOOTER_COLOR y CUSTOM_PAGE_FOOTER_CONTENT.

Las variables de entorno (CUSTOM_LOGIN_LOGO, CUSTOM_EULA_POLICY, CUSTOM_PAGE_HEADER_COLOR, CUSTOM_PAGE_HEADER_CONTENT, CUSTOM_PAGE_FOOTER_COLOR, CUSTOM_PAGE_FOOTER_CONTENT) se pueden definir en el archivo values.yaml en el chart de helm. La sección correspondiente en el archivo values.yaml donde se pueden definir estas variables es "manager.env.envs"

4. Ejemplo para personalizar las páginas de la interfaz de usuario utilizando el chart de helm

Requisitos previos

Antes de personalizar el componente de la interfaz de usuario, asegúrate de tener lo siguiente:

  • SUSE® Security Repositorio del chart de Helm: https://github.com/neuvector/neuvector-helm

  • SUSE® Security instalado en tu clúster de Kubernetes

Pasos

Siga estos pasos para personalizar el componente de la interfaz de usuario:

  1. Localiza el repositorio del chart de Helm SUSE® Security en tu entorno local: https://github.com/neuvector/neuvector-helm.

  2. Desplácese al directorio neuvector-helm/charts/core.

  3. Edita el manager.env.envs en el values.yaml para agregar las variables de entorno.

    • CUSTOM_PAGE_HEADER_COLOR: Establezca el color del banner en DarkOrange (#ff8c00).

    • CUSTOM_PAGE_HEADER_CONTENT: Especifica el texto del encabezado utilizando HTML o texto plano, por ejemplo "Estás accediendo a un U.S. Sistema de Información del Gobierno (USG) que se proporciona únicamente para uso autorizado por el USG.", recuerda codificarlo en base64.

    • CUSTOM_EULA_POLICY: Personaliza el contenido de la política utilizando un fragmento de HTML simple o texto y recuerda codificarlo en base64.

    • CUSTOM_LOGIN_LOGO: Personaliza el logo con un archivo svg de 300x80 píxeles y codifícalo en base64.

5. Ejemplo para personalizar las páginas de la interfaz de usuario utilizando manifiestos.

Requisitos previos

Antes de personalizar el componente de la interfaz de usuario, asegúrate de tener lo siguiente:

  • Repositorio de manifiestos SUSE® Security: https://github.com/neuvector/manifests

  • SUSE® Security instalado en su clúster de Kubernetes

Pasos

Sigue estos pasos para personalizar el componente de la interfaz de usuario:

  1. Localiza el repositorio de manifiestos SUSE® Security en tu entorno local: https://github.com/neuvector/manifests.

  2. Desplázate al directorio manifests/kubernetes.

  3. Elige el manifiesto adecuado según tu entorno. Por ejemplo, elige neuvector-containerd-k8s.yaml para un clúster k8s con containerd entorno de ejecución de contenedor.

  4. Localiza la sección de despliegue neuvector-manager-pod en el archivo de manifiesto para añadir las variables de entorno:

    • CUSTOM_PAGE_HEADER_COLOR: Establece el color del banner en DarkOrange (#ff8c00).

    • CUSTOM_PAGE_HEADER_CONTENT: Especifica el texto del encabezado utilizando HTML o texto plano, por ejemplo "Estás accediendo a un U.S. Sistema de Información del Gobierno (USG) que se proporciona únicamente para uso autorizado por el USG.", recuerda codificarlo en base64.

    • CUSTOM_EULA_POLICY: Personaliza el contenido de la directiva utilizando un fragmento de HTML simple o texto y recuerda codificarlo en base64.

    • CUSTOM_LOGIN_LOGO: Personaliza el logo con un archivo svg de 300x80 píxeles y codifícalo en base64.
