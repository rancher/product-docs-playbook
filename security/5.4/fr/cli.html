Ligne de commande

Utilisation de la ligne de commande SUSE® Security

La SUSE® Security solution prend en charge un ensemble limité de fonctions via l’interface de ligne de commande (CLI). La CLI est prise en charge par le Gestionnaire, qui utilise à son tour une API REST pour émettre des commandes au Contrôleur. Le Contrôleur gère ensuite les Enforceurs de manière appropriée. Un ensemble complet d’opérations est pris en charge via l’API REST, qui peut être exposée directement depuis le Contrôleur. Vous pouvez accéder à la SUSE® Security CLI en tapant la commande CLI pour le Gestionnaire ou Allinone, par exemple :

kubectl exec -it neuvector-manager-pod-5bb76b6754-rlmnp -n neuvector -- cli
docker exec -it allinone cli

Où ‘allinone’ est le nom du conteneur pour le Contrôleur. Vous devrez peut-être utiliser l’ID du conteneur pour le nom.

Bien que la CLI soit disponible via le Gestionnaire, nous recommandons d’utiliser directement l’API REST dans le Contrôleur pour les requêtes et l’automatisation.

Renifleur de paquets

Les fichiers de renifleur sont stockés dans le répertoire /var/neuvector/pcap dans le conteneur Enforceur. Assurez-vous de mapper le volume à votre répertoire de machine invité ou à votre répertoire système local pour pouvoir accéder aux fichiers. Par exemple, dans le fichier docker-compose, ajoutez ‘- /var/neuvector:/var/neuvector’ dans volumes.

Pour commencer la capture de paquets sur un pod, vous devrez connaître l’ID du conteneur à passer dans le champ ID_OR_NAME. Vous pouvez le faire avec show container -c <container_name>. Ensuite, démarrez le renifleur avec request sniffer start <container_id>. Exemples :
