カスタマイズされたログイン、ヘッダーおよびフッター

コンソール（UI）コンポーネントのカスタマイズ

このガイドでは、次のオプションを使用してコンソールをカスタマイズする方法を説明します：

1.ログインロゴのカスタマイズ

ログインロゴをカスタマイズするには、次の手順に従ってください：

  • 300x80ピクセルのSVGファイルを作成します。

  • SVGファイルをBase64エンコードします。

  • エンコードされたファイルを環境変数`CUSTOM_LOGIN_LOGO`に保存します。

2.受け入れポリシーのカスタマイズ（例：利用規約）

ポリシーをカスタマイズするには、次の手順に従ってください：

  • ポリシーの内容は、プレーンHTMLまたはテキストにすることができます。

  • ポリシーの内容をBase64エンコードします。

  • エンコードされた内容を環境変数`CUSTOM_EULA_POLICY`に保存します。

3.ページバナーのカスタマイズ（すべてのページに表示）

ページバナーをカスタマイズするには、次の手順に従ってください：

ヘッダーのカスタマイズ

  • ヘッダーの内容と色をカスタマイズします。

  • ヘッダーバナーの色は必須です。

  • 色の値は、色のキーワード（例：黄色）または16進数の値（例：#ffff00）で指定できます。

  • 内容はオプションで、最大120文字のプレーンHTMLまたはテキストの1行にすることができます。

  • ヘッダーの内容をBase64エンコードします。

  • エンコードされた内容を環境変数`CUSTOM_PAGE_HEADER_COLOR`と`CUSTOM_PAGE_HEADER_CONTENT`に保存します。

フッターのカスタマイズ

  • フッターの内容と色をカスタマイズします。

  • フッターバナーの色は、色がカスタマイズされていない場合、ヘッダーバナーと同じになります。

  • 内容はオプションで、最大120文字のプレーンHTMLまたはテキストの1行にすることができます。

  • フッターの内容をBase64エンコードします。

  • エンコードされた内容を環境変数`CUSTOM_PAGE_FOOTER_COLOR`と`CUSTOM_PAGE_FOOTER_CONTENT`に保存します。

環境変数（CUSTOM_LOGIN_LOGO、CUSTOM_EULA_POLICY、CUSTOM_PAGE_HEADER_COLOR、CUSTOM_PAGE_HEADER_CONTENT、CUSTOM_PAGE_FOOTER_COLOR、CUSTOM_PAGE_FOOTER_CONTENT）は、 helm chartの`values.yaml`ファイルに定義できます。これらの変数を定義できる`values.yaml`ファイルの対応するセクションは「manager.env.envs」です。

4.Helmチャートを使用してUIページをカスタマイズする例

前提条件

UIコンポーネントをカスタマイズする前に、次のことを確認してください：

  • SUSE® Security Helmチャートリポジトリ： https://github.com/neuvector/neuvector-helm

  • SUSE® SecurityがKubernetesクラスターにインストールされています。

手順

UIコンポーネントをカスタマイズするための手順は次のとおりです：

  1. ローカル環境でSUSE® Security Helmチャートリポジトリを見つけます： https://github.com/neuvector/neuvector-helm.

  2. `neuvector-helm/charts/core`ディレクトリに移動します。

  3. `values.yaml`の`manager.env.envs`を編集して、環境変数を追加します。

    • CUSTOM_PAGE_HEADER_COLOR:バナーの色をDarkOrange（#ff8c00）に設定します。

    • CUSTOM_PAGE_HEADER_CONTENT:ヘッダーテキストをプレーンなHTMLまたはテキストで指定します。例えば、「あなたは米国…政府（USG）情報システム（IS）は、USG認可の使用専用として提供されています。」を忘れずにBase64エンコードしてください。

    • CUSTOM_EULA_POLICY:ポリシーの内容をプレーンなHTMLスニペットまたはテキストを使用してカスタマイズし、base64エンコードすることを忘れないでください。

    • CUSTOM_LOGIN_LOGO:ロゴを300x80ピクセルのsvgファイルでカスタマイズし、base64エンコードしてください。

5.マニフェストを使用してUIページをカスタマイズする例

前提条件

UIコンポーネントをカスタマイズする前に、次のことを確認してください：

  • SUSE® Securityマニフェストリポジトリ: https://github.com/neuvector/manifests

  • SUSE® SecurityがKubernetesクラスターにインストールされています。

手順

UIコンポーネントをカスタマイズするための手順は次のとおりです：

  1. ローカル環境でSUSE® Securityマニフェストリポジトリを見つけてください: https://github.com/neuvector/manifests.

  2. `manifests/kubernetes`ディレクトリに移動します。

  3. 環境に応じて適切なマニフェストを選択してください。例えば、`containerd`コンテナランタイムを持つk8sクラスターには`neuvector-containerd-k8s.yaml`を選択してください。

  4. 環境変数を追加するために、マニフェストファイル内の`neuvector-manager-pod`デプロイメントセクションを見つけてください:

    • CUSTOM_PAGE_HEADER_COLOR:バナーの色をDarkOrange（#ff8c00）に設定します。

    • CUSTOM_PAGE_HEADER_CONTENT:ヘッダーテキストをプレーンなHTMLまたはテキストで指定します。例えば、「あなたは米国の情報システム（IS）にアクセスしています…政府（USG）情報システム（IS）は、USG認可の使用専用として提供されています。」を忘れずにBase64エンコードしてください。

    • CUSTOM_EULA_POLICY:ポリシーの内容をプレーンなHTMLスニペットまたはテキストを使用してカスタマイズし、base64エンコードすることを忘れないでください。
