Rancher vCluster (Experimentell)

rancher-vcluster ist ein experimentelles Add-on. Es ist nicht im ISO enthalten, aber Sie können es aus dem experimental-addons Repository herunterladen. Für weitere Informationen zu experimentellen Funktionen siehe Funktionsbezeichnungen.

Das rancher-vcluster Add-on ermöglicht es Ihnen, Rancher als Arbeitslast im zugrunde liegenden SUSE Virtualization Cluster auszuführen und wird mit vCluster implementiert.

Das Add-on führt ein geschachteltes K3s-Cluster im rancher-vcluster Namespace aus und stellt Rancher in diesem Cluster bereit.

Während der Installation wird der Ingress für Rancher mit dem SUSE Virtualization Cluster synchronisiert, sodass Endbenutzer auf Rancher zugreifen können.

Installieren des Add-ons

Wenn Sie die SUSE Virtualization kubeconfig-Datei verwenden, können Sie das Add-on installieren, indem Sie den folgenden Befehl ausführen:

kubectl apply -f https://raw.githubusercontent.com/harvester/experimental-addons/main/rancher-vcluster/rancher-vcluster.yaml

Konfigurieren des Add-ons

Nach der Installation konfigurieren Sie das Add-on über die SUSE Virtualization UI.

  1. Gehen Sie zu Erweiterte → Add-ons.

  2. Suchen Sie das rancher-vcluster Add-on und wählen Sie dann ⋮ → Konfiguration bearbeiten aus.

  3. Geben Sie im Feld Hostname einen gültigen DNS-Eintrag ein, der auf die SUSE Virtualization VIP zeigt. Dies ist wichtig, da der vcluster Ingress mit dem übergeordneten SUSE Virtualization Cluster synchronisiert wird. Ein gültiger Hostname wird verwendet, um den Ingress-Verkehr zur vcluster-Arbeitslast zu filtern.

  4. Geben Sie im Feld Bootstrap-Passwort das Bootstrap-Passwort für das neue Rancher ein, das auf dem vcluster bereitgestellt wird.

Sobald das Add-on bereitgestellt ist, kann es einige Minuten dauern, bis Rancher verfügbar ist.

Sie können dann auf Rancher über den von Ihnen bereitgestellten Hostnamen-DNS-Eintrag zugreifen.

Siehe Rancher Integration für weitere Informationen.

Das Add-on wird auf einem vcluster Statefulset bereitgestellt, das ein Longhorn PVC mit dem Namen data-rancher-vcluster-0 verwendet.

Wenn Sie das Add-on deaktivieren, bleibt das PVC im rancher-vcluster Namespace. Wenn Sie das Add-on erneut aktivieren, wird das PVC wiederverwendet und der vorherige Zustand von Rancher wird wiederhergestellt.

Wenn Sie die Daten löschen möchten, stellen Sie sicher, dass das PVC gelöscht wird.
