Glossar

Cluster-Netzwerk

Verkehrsisolierter Weiterleitungsweg für die Übertragung von Netzwerkverkehr im SUSE Virtualization Cluster.

Gast-Cluster / Gast-Kubernetes-Cluster

Gruppe integrierter virtueller Maschinen, die auf einem SUSE Virtualization Cluster laufen.

Sie können RKE2- und K3s-Gast-Cluster über die SUSE Virtualization und SUSE Rancher Prime Schnittstellen erstellen. Die Erstellung von Gast-Clustern umfasst das Abrufen von Images aus dem Internet oder einer privaten Registry.

Gast-Cluster bilden die Hauptinfrastruktur für das Ausführen von Container-Workloads. Bestimmte Versionen von SUSE Virtualization und SUSE Rancher Prime ermöglichen es Ihnen, Container-Workloads direkt auf SUSE Virtualization Clustern bereitzustellen (mit einigen Einschränkungen).

Gast-Knoten / Gast-Cluster-Knoten

Virtuelle Maschine, die die Ressourcen des SUSE Virtualization Clusters nutzt, um Container-Workloads auszuführen.

Gast-Knoten werden über eine Steuerungsebene verwaltet, die die podbezogene Aktivität steuert und den gewünschten Clusterzustand aufrechterhält.

SUSE Virtualization Cluster

Gruppe integrierter physischer Server (Hosts), auf denen der SUSE Virtualization Hypervisor installiert ist. Diese Server verwalten gemeinsam Rechenressourcen, Arbeitsspeicher und Speicherressourcen, um eine Umgebung für den Betrieb virtueller Maschinen bereitzustellen.

Ein Cluster mit drei Knoten ist erforderlich, um die Multi-Node-Funktionen von SUSE Virtualization vollständig zu realisieren, insbesondere die hohe Verfügbarkeit. Die neuesten Versionen ermöglichen es Ihnen, Cluster mit zwei Verwaltungsknoten und einem Zeugenknoten zu erstellen (und optional einen oder mehrere Arbeitsknoten). Sie können auch Einzelknoten-Cluster erstellen, die die meisten Funktionen unterstützen (außer hoher Verfügbarkeit, Multi-Replikat-Unterstützung und Live-Migration).

SUSE Virtualization Cluster können in SUSE Rancher Prime importiert und verwaltet werden. In diesem Kontext wird ein importierter SUSE Virtualization Cluster als "verwalteter Cluster" oder "Downstream-Benutzer-Cluster" bezeichnet (häufig abgekürzt als "Downstream-Cluster"). Der Begriff bezieht sich auf jeden Kubernetes-Cluster, der mit einem SUSE Rancher Prime Server verbunden ist.

Bestimmte Versionen von SUSE Virtualization und SUSE Rancher Prime ermöglichen es Ihnen, Container-Workloads direkt auf SUSE Virtualization Clustern bereitzustellen (mit einigen Einschränkungen). Wenn dieses experimentelles Feature aktiviert ist, interagieren Container-Workloads nahtlos mit virtuellen Maschinen-Workloads.

SUSE Virtualization Hypervisor

Spezialisiertes Betriebssystem und Software-Stack, das auf einem einzelnen physischen Server läuft.

SUSE Virtualization ISO

Installationsimage, das die Kernkomponenten des Betriebssystems und alle erforderlichen Container-Images enthält, die während der Installation vorab geladen werden.

SUSE Virtualization Knoten

Physischer Server, auf dem der SUSE Virtualization Hypervisor installiert ist.

Jeder Knoten, der einem SUSE Virtualization Cluster beitritt, muss eine Rolle zugewiesen bekommen, die die Funktionen bestimmt, die der Knoten innerhalb des Clusters ausführen kann. Alle SUSE Virtualization Knoten verarbeiten Daten, aber nicht alle können Daten speichern.

Harvester Cloud Provider

Komponente, die es einem Rancher-verwalteten SUSE Virtualization Cluster ermöglicht, als native Cloud-Backend für Gast-Kubernetes-Cluster zu fungieren. Der Harvester Cloud Provider integriert den Kubernetes Cloud Controller Manager (CCM) und den Container Storage Interface (CSI) Treiber, sodass Gast-Cluster dynamisch SUSE Virtualization-native Ressourcen (Lastenausgleich und Speicher) anfordern und verwalten können, ohne manuelles Eingreifen.

Harvester CSI Treiber

Treiber, der eine standardisierte Container Storage Interface (CSI) für Gast-Kubernetes-Cluster bereitstellt, sodass Container-Workloads die Speicherlösung nutzen können, die mit dem zugrunde liegenden SUSE Virtualization Cluster integriert ist. Diese Komponente ermöglicht das Hot-Plugging von Volumes zu den virtuellen Maschinen, um native Speicherleistung bereitzustellen.

Harvester Node Treiber

Treiber, den SUSE Rancher Prime verwendet, um virtuelle Maschinen in einem SUSE Virtualization Cluster bereitzustellen und um Gast-Kubernetes-Cluster auf diesen virtuellen Maschinen zu starten und zu verwalten.

Live-Migration

Prozess des Verschiebens einer laufenden virtuellen Maschine zu einem anderen Knoten innerhalb desselben SUSE Virtualization Clusters, ohne das Gastbetriebssystem zu unterbrechen und Ausfallzeiten der Workloads zu verursachen. Live-Migration kann nur erfolgen, wenn die Voraussetzungen erfüllt sind und wenn die betroffenen virtuellen Maschinen live-migrierbar sind.

mgmt

Cluster-Netzwerk, das während der SUSE Virtualization Cluster-Bereitstellung automatisch erstellt wird und auf allen Hosts immer aktiviert ist. SUSE Virtualization verwendet mgmt für die Kommunikation innerhalb des Clusters und für Cluster-Management-Aufgaben.

Netzinstallations-ISO

Installationsabbild, das nur die Kern SUSE Virtualization Betriebssystemkomponenten enthält, sodass der Installer booten und dann das Betriebssystem auf einer Festplatte installieren kann. Nach Abschluss der Installation zieht das Betriebssystem alle erforderlichen Container-Images aus dem Internet.

Netzwerkkonfiguration

Definition, wie eine Gruppe von Clusterknoten mit einheitlichen Netzwerkspezifikationen sich mit einem bestimmten Cluster-Netzwerk verbindet.

Overlay-Netzwerk

Virtuelles Netzwerk, das einen virtuellen Layer-2-Switch darstellt, der Verkehr zwischen virtuellen Maschinen kapselt und weiterleitet. Overlay-Netzwerke unterstützen fortschrittliche Software-Defined Networking (SDN) Funktionen wie virtuelle private Clouds (VPCs) und Subnetze für Arbeitslasten von virtuellen Maschinen.

Speichernetzwerk

Netzwerk zur Isolierung des Longhorn-Replikationstraffics vom Intra-Cluster-Traffic auf mgmt und anderen clusterweiten Arbeitslasten.

VLAN-Trunk-Netzwerk

Virtuelles Netzwerk, das einer virtuellen Maschine eine einzelne Netzwerkschnittstelle bereitstellt, die gleichzeitig Verkehr für mehrere, sich überschneidende VLAN-ID-Bereiche transportieren kann. Wenn eine virtuelle Maschine an ein VLAN-Trunk-Netzwerk angeschlossen ist, dürfen das Gastbetriebssystem und die Anwendungen Pakete senden und empfangen, die mit einer der VLAN-IDs innerhalb des angegebenen Bereichs gekennzeichnet sind.

VM-Migrationsnetzwerk

Netzwerk zur Isolierung des Verkehrs für die Migration virtueller Maschinen vom Intra-Cluster-Traffic auf mgmt und anderen clusterweiten Arbeitslasten.

VM-Netzwerk

Virtuelles Netzwerk, das mit einem bestimmten Cluster-Netzwerk verbunden ist und die Kommunikation zwischen virtuellen Maschinen und dem externen Netzwerk ermöglicht.

Zeugenknoten

Nicht-rechnender Knoten wird ausschließlich zur Aufrechterhaltung des Cluster-Konsenses verwendet. Es stellt sicher, dass das System eine Mehrheitsentscheidung (Quorum) über Cluster-Updates treffen kann, selbst wenn ein Verwaltungs-Knoten nicht verfügbar wird oder ein Netzwerkfehler auftritt.

Zeugenknoten führen keine Arbeitslasten aus und speichern keine Daten. Jeder SUSE Virtualization Cluster kann nur einen Zeugenknoten haben.
