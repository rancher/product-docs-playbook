Verwaltungsadresse

SUSE Virtualization stellt eine feste virtuelle IP (VIP) als Verwaltungsadresse bereit, wobei die VIP sich von jeder Knoten-IP unterscheiden muss. Die Verwaltungsadresse finden Sie im Dashboard der Konsole nach der Installation.

Wenn Sie die IP-Adresse ausgewählt haben, die über DHCP konfiguriert werden soll, müssen Sie eine statische MAC-zu-IP-Adresszuordnung auf Ihrem DHCP-Server konfigurieren, um eine persistente virtuelle IP zu haben.

Anforderungen

  • Die VIP und die Management-Schnittstellen der Knoten müssen im selben Subnetz liegen.

  • Alle SUSE Virtualization Management-Schnittstellen der Knoten müssen sich im selben Layer-2-Netzwerksegment befinden.

Beide sind erforderlich, da die VIP auf dem Address Resolution Protocol (ARP) basiert, das ein Layer-2-Protokoll ist.

Die VIP kann jeder SUSE Virtualization Management-Schnittstelle der Knoten zugewiesen werden und kann jederzeit wechseln (nicht nur bei einem Knotenfehler). Wenn die VIP die Hosts wechselt, werden gratuitous ARPs gesendet, damit andere Hosts im Netzwerk wissen, wohin sie den Verkehr für die Verwaltungs-IP-Adresse des SUSE Virtualization Clusters (die VIP) leiten sollen.

Wenn Sie planen, einen SUSE Virtualization Cluster in einem Bare-Metal-Rechenzentrum zu hosten, wird der Dienstanbieter wahrscheinlich eine "schwebende" oder "reservierte" IP-Adresse als VIP festlegen, die Sie einem Ihrer Server zuweisen können. Das Aktualisieren dieser Zuordnung bewirkt, dass der Verkehr sofort zu dem neuen Knoten fließt, dem er zugewiesen ist. In diesen Situationen kann SUSE Virtualization die schwebende/reservierte IP-Zuordnung in den Systemen Ihres Anbieters nicht aktualisieren, wenn die VIP die Hosts wechselt.

Darüber hinaus können die "gratuitous ARPs" der VIP je nach Netzwerkkonfiguration Ihres Anbieters zwischen Ihren SUSE Virtualization Knoten ebenfalls ineffektiv sein (zum Beispiel, wenn Ihre SUSE Virtualization Hosts sich nicht im selben Layer-2-Netzwerk befinden). Sie müssen die schwebende/reservierte IP-Zuordnung manuell aktualisieren, wenn die VIP die Hosts wechselt, um sicherzustellen, dass der Cluster ordnungsgemäß funktioniert. Für weitere Informationen siehe Identifizierung des Knotens, dem die VIP zugewiesen ist.

Wie man die VIP-MAC-Adresse erhält

Um die VIP-MAC-Adresse zu erhalten, können Sie den folgenden Befehl auf dem Verwaltungs-Knoten ausführen:

$ kubectl get svc -n kube-system ingress-expose -ojsonpath='{.metadata.annotations.kube-vip\.io/hwaddr}'

Beispiel für die Ausgabe:

{"kube-vip.io/hwaddr":"02:00:00:09:7f:3f","kube-vip.io/requestedIP":"10.84.102.31"}

Identifizierung des Knotens, dem die VIP zugewiesen ist

Sie können kubectl verwenden (entweder auf Ihrem lokalen Rechner mit der SUSE Virtualization kubeconfig-Datei oder beim Verwenden von SSH, um sich mit einem beliebigen SUSE Virtualization Verwaltungs-Knoten als Root-Benutzer zu verbinden).

kubectl -n kube-system get svc ingress-expose -o jsonpath='{.metadata.annotations.kube-vip\.io/vipHost}'

Beispielausgabe:

harvester-xzj76

Alternativ können Sie SSH verwenden, um sich mit jedem SUSE Virtualization Verwaltungs-Knoten zu verbinden und dann den Befehl ip address show mgmt-br auszuführen.

Beispiel:

harvester-xzj76:~ # ip address show mgmt-br
4: mgmt-br: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc noqueue state UP group default qlen 1000
    link/ether 8c:dc:d4:b5:f0:fd brd ff:ff:ff:ff:ff:ff
    inet 172.19.108.45/21 brd 172.19.111.255 scope global mgmt-br
       valid_lft forever preferred_lft forever
    inet 172.19.108.34/32 scope global mgmt-br
       valid_lft forever preferred_lft forever

Wenn die Ausgabe sowohl die VIP- als auch die Knotenadresse enthält, wird die VIP dem Knoten zugewiesen, mit dem Sie verbunden sind.

Verwendungen

Die Verwaltungsadresse:

  • Ermöglicht den Zugriff auf die SUSE Virtualization API/UI über das HTTPS Protokoll.

  • Ermöglicht anderen Knoten, dem Cluster beizutreten.
