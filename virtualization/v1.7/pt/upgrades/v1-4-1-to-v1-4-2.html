Fazer upgrade da v1.4.1 para v1.4.2

Informações gerais

Um botão Fazer upgrade aparece na tela Dashboard sempre que uma nova versão SUSE Virtualization para a qual você pode fazer upgrade se torna disponível. Para mais informações, veja Iniciar um upgrade.

Para ambientes air-gapped, veja Preparar um upgrade air-gapped.

Atualizar a extensão da UI do Harvester para SUSE Rancher Prime v2.10.1

Para importar clusters SUSE Virtualization v1.4.2 no Rancher v2.10.1, você deve usar v1.0.3 da extensão da UI Rancher para SUSE Virtualization.

  1. Na UI do Rancher, vá para local → Apps → Repositórios.

  2. Localize o repositório chamado harvester, e então selecione ⋮ → Atualizar.

    Este repositório possui as seguintes propriedades:

    • URL: https://github.com/harvester/harvester-ui-extension

    • Branch: gh-pages

  3. Vá para a tela Extensões.

  4. Localize a extensão chamada Harvester, e então clique em Atualizar.

  5. Selecione a versão 1.0.3, e então clique em Atualizar.

  6. Permita algum tempo para que a extensão seja atualizada, e então atualize a tela.

A UI do Rancher exibe uma mensagem de erro após a atualização da extensão. A mensagem de erro desaparece quando você atualiza a tela.

Este problema, que existe nas versões Rancher v2.10.0 e v2.10.1, será corrigido na v2.10.2.

Problemas relacionados: #7234, #107

Compatibilidade de backup da máquina virtual

Você pode encontrar certas limitações ao criar e restaurar backups que envolvem armazenamento externo.

Problemas conhecidos

Uso elevado de CPU

O uso elevado de CPU pode ocorrer devido ao campo backup-target da configuração refreshIntervalInSeconds. Se o campo estiver vazio ou definido como 0, SUSE Virtualization atualiza constantemente o alvo de backup, resultando em alto uso de CPU.

Para corrigir o problema, atualize o valor de refreshIntervalInSeconds para um número maior (por exemplo, 60) usando o comando kubectl edit setting backup-target.

Exemplo:

value: '{"type":"nfs","endpoint":"nfs://longhorn-test-nfs-svc.default:/opt/backupstore", "refreshIntervalInSeconds": 60}'

Problema relacionado: #7885

O upgrade reinicia inesperadamente após clicar no botão "Desconsiderar".

Quando você usa Rancher para fazer upgrade de SUSE Virtualization, a UI Rancher exibe uma caixa de diálogo com um botão rotulado "Desconsiderar". Clicar neste botão pode resultar nos seguintes problemas:

  • A seção status do CR harvesterhci.io/v1beta1/upgrade é limpa, causando a perda de todas as informações importantes sobre o upgrade.

  • O processo de upgrade reinicia inesperadamente.

Esse problema afeta Rancher v2.10.x, que usa v1.0.2, v1.0.3 e v1.0.4 da Extensão da UI do Harvester. Todas as versões da UI SUSE Virtualization não são afetadas. O problema foi corrigido na Extensão da UI do Harvester v1.0.5 e v1.5.0.

Para evitar esse problema, execute uma das seguintes ações:

  • Use a UI SUSE Virtualization para fazer upgrade. Clicar no botão "Desconsiderar" na UI SUSE Virtualization não resulta em comportamento inesperado.

  • Em vez de clicar no botão na UI Rancher, execute o seguinte comando contra o cluster:

    kubectl -n harvester-system label upgrades -l harvesterhci.io/latestUpgrade=true harvesterhci.io/read-message=true

Problema relacionado: #7791

Máquinas virtuais que usam volumes RWX migráveis reiniciam inesperadamente

Máquinas virtuais que usam RWX volumes migráveis reiniciam inesperadamente quando os pods do plugin CSI são reiniciados. Esse problema afeta SUSE Virtualization v1.4.x, v1.5.0 e v1.5.1.

A solução alternativa é desativar a configuração Excluir Automaticamente o Pod de Trabalho Quando o Volume for Desconectado Inesperadamente na UI SUSE Storage antes de iniciar o upgrade. Você deve habilitar a configuração novamente assim que o upgrade for concluído.

O problema será corrigido na versão SUSE Storage v1.8.3, v1.9.1 e versões posteriores. SUSE Virtualization v1.6.0 incluirá SUSE Storage v1.9.1.
