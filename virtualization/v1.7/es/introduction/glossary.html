Glosario

red de clúster

Vía de reenvío aislada por tráfico para la transmisión de tráfico de red en el SUSE Virtualization clúster.

clúster de invitados / clúster de Kubernetes de invitados

Grupo de máquinas virtuales integradas que se ejecutan sobre un SUSE Virtualization clúster.

Puedes crear clústeres de invitados RKE2 y K3s utilizando las interfaces SUSE Virtualization y SUSE Rancher Prime. La creación de clústeres de invitados implica obtener imágenes de internet o de un registro privado.

Los clústeres de invitados forman la infraestructura principal para ejecutar cargas de trabajo en contenedores. Ciertas versiones de SUSE Virtualization y SUSE Rancher Prime te permiten desplegar cargas de trabajo en contenedores directamente en SUSE Virtualization clústeres (con algunas limitaciones).

nodo de invitados / nodo de clúster de invitados

Máquina virtual que utiliza los recursos del SUSE Virtualization clúster para ejecutar cargas de trabajo en contenedores.

Los nodos de invitados se gestionan a través de un plano de control que controla la actividad relacionada con los pods y mantiene el estado deseado del clúster.

SUSE Virtualization clúster

Grupo de servidores físicos integrados (hosts) en los que está instalado el SUSE Virtualization hipervisor. Estos servidores gestionan colectivamente los recursos de computación, memoria y almacenamiento para proporcionar un entorno para ejecutar máquinas virtuales.

Se requiere un clúster de tres nodos para realizar plenamente las características de múltiples nodos de SUSE Virtualization, particularmente la alta disponibilidad. Las versiones más recientes te permiten crear clústeres con dos nodos de gestión y un nodo testigo (y opcionalmente, uno o más nodos de trabajo). También puedes crear clústeres de un solo nodo que soportan la mayoría de las características (excluyendo alta disponibilidad, soporte multi-replica y migración en vivo).

Los clústeres SUSE Virtualization pueden ser importados y gestionados por SUSE Rancher Prime. Dentro de este contexto, un clúster SUSE Virtualization importado se conoce como "clúster downstream" o "clúster de usuario descendente" (a menudo abreviado como "clúster descendente"). El término se refiere a cualquier clúster de Kubernetes que esté conectado a un SUSE Rancher Prime servidor.

Ciertas versiones de SUSE Virtualization y SUSE Rancher Prime permiten desplegar cargas de trabajo en contenedores directamente en clústeres SUSE Virtualization (con algunas limitaciones). Cuando esta característica experimental está habilitada, las cargas de trabajo en contenedores interactúan sin problemas con las cargas de trabajo de máquinas virtuales.

SUSE Virtualization hipervisor

Sistema operativo especializado y conjunto de software que se ejecuta en un único servidor físico.

SUSE Virtualization ISO

Imagen de instalación que contiene los componentes del kernel del sistema operativo y todas las imágenes de contenedor requeridas, que se precargan durante la instalación.

SUSE Virtualization nodo

Servidor físico en el que está instalado el SUSE Virtualization hipervisor.

Cada nodo que se une a un SUSE Virtualization clúster debe ser asignado a un rol que determina las funciones que el nodo puede realizar dentro del clúster. Todos los SUSE Virtualization nodos procesan datos, pero no todos pueden almacenar datos.

Proveedor de Nube Harvester

Componente que permite que un clúster downstream (SUSE Virtualization), gestionado por Rancher, funcione como un backend nativo de nube para clústeres Kubernetes invitados. El Proveedor de Nube Harvester integra el administrador de controladores de nube de Kubernetes (CCM) y el controlador de interfaz de almacenamiento de contenedores (CSI), permitiendo que los clústeres invitados soliciten y gestionen dinámicamente recursos nativos de SUSE Virtualization (balanceador de carga y almacenamiento) sin intervención manual.

Controlador CSI de Harvester

Controlador que proporciona una interfaz estándar de almacenamiento de contenedores (CSI) para clústeres Kubernetes invitados, permitiendo que las cargas de trabajo en contenedores utilicen la solución de almacenamiento integrada con el SUSE Virtualization clúster subyacente. Este componente permite la conexión en caliente de volúmenes a las máquinas virtuales para proporcionar un rendimiento de almacenamiento nativo.

Controlador de Nodo Harvester

Controlador que SUSE Rancher Prime utiliza para provisionar máquinas virtuales en un SUSE Virtualization clúster, y para lanzar y gestionar clústeres Kubernetes invitados sobre esas máquinas virtuales.

migración en vivo

Proceso de mover una máquina virtual en ejecución a otro nodo dentro del mismo SUSE Virtualization clúster sin interrumpir el sistema operativo invitado y causar tiempo de inactividad en la carga de trabajo. La migración en vivo solo puede ocurrir cuando se cumplen los requisitos previos y cuando las máquinas virtuales afectadas son migrables en vivo.

gestión

Red de clúster que se crea automáticamente durante la ampliación del SUSE Virtualization clúster y que siempre está habilitada en todos los hosts. SUSE Virtualization utiliza mgmt para las comunicaciones intra-clúster y las tareas de gestión del clúster.

ISO de instalación de red

Imagen de instalación que contiene solo los componentes del kernel SUSE Virtualization del sistema operativo, permitiendo que el instalador arranque y luego instale el sistema operativo en un disco. Después de que se completa la instalación, el sistema operativo descarga todas las imágenes de contenedor requeridas de internet.

configuración de red

Definición de cómo un conjunto de nodos de clúster con especificaciones de red uniformes se conecta a una red de clúster específica.

red de superposición

Red virtual que representa un conmutador virtual de capa 2 que encapsula y reenvía el tráfico entre máquinas virtuales. Las redes de superposición admiten capacidades avanzadas de redes definidas por software (SDN) como nubes privadas virtuales (VPC) y subredes para cargas de trabajo de máquinas virtuales.

red de almacenamiento

Red para aislar el tráfico de replicación de Longhorn del tráfico intra-clúster en mgmt y otras cargas de trabajo a nivel de clúster.

red troncal VLAN

Red virtual que proporciona a una máquina virtual una única interfaz de red que puede transportar tráfico para varios rangos de ID de VLAN superpuestos simultáneamente. Cuando una máquina virtual está conectada a una red troncal VLAN, se permite al sistema operativo invitado y a las aplicaciones enviar y recibir paquetes etiquetados con cualquiera de los ID de VLAN dentro del rango especificado.

red de migración de VM

Red para aislar el tráfico de migración de máquinas virtuales del tráfico intra-clúster en mgmt y otras cargas de trabajo a nivel de clúster.

red de VM

Red virtual vinculada a una red de clúster específica que permite la comunicación entre máquinas virtuales y la red externa.

nodo testigo

nodo no computacional utilizado únicamente para mantener el consenso del clúster. Asegura que el sistema pueda alcanzar una decisión mayoritaria (quórum) sobre las actualizaciones del clúster, incluso si un nodo de gestión se vuelve no disponible o ocurre una falla de red.

Los nodos testigos no ejecutan cargas de trabajo ni almacenan datos. Cada SUSE Virtualization clúster puede tener solo un nodo testigo.
