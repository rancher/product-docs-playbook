Importar un clúster construido sobre una VM de SUSE® Virtualization

Rancher te permite importar VMs de SUSE® Virtualization en las que instalaste Kubernetes.

Distribución

Requisitos previos

  • El clúster de Kubernetes se construye sobre VMs de SUSE® Virtualization.

  2. Genera la configuración cloud-config del proveedor de nube utilizando el guion generate_addon.sh, que está disponible en el repositorio harvester/cloud-provider-harvester.

    Ejemplo:

     ./generate_addon.sh <serviceaccount name> <namespace>

    La salida generada será similar a la siguiente: La salida será como sigue:

     ########## cloud config ############
 apiVersion: v1

     cluster: local

   content: <CONTENT>
   owner: root:root
   path: /etc/kubernetes/cloud-config
   permissions: '0644'

  3. Crea una VM con dos archivos de configuración cloud-config.

  4. Instala RKE2 en la VM.

     sudo mkdir -p /etc/rancher/rke2
 echo "cni: calico
 disable-kube-proxy: false
 etcd-expose-metrics: false" | sudo tee /etc/rancher/rke2/config.yaml
 curl -sfL https://get.rke2.io | sudo sh -
 sudo systemctl enable rke2-server.service
 sudo systemctl start rke2-server.service

  5. Verifica que RKE2 esté en funcionamiento en la VM.

     sudo /var/lib/rancher/rke2/bin/kubectl --kubeconfig /etc/rancher/rke2/rke2.yaml get nodes

  6. Importa la VM en Rancher.

    En la interfaz de Rancher, ve a Gestión de Clústeres > Clústeres > Importar Existente > Genérico > Crear.

     # Run the command in the VM
 curl --insecure -sfL https://192.168.0.181:6443/v3/import/g5p2g2gtxw4564nktdl4nr5cwwvtwqp9zxd6dmhm5nc7vpnxmr9cfk_c-m-mzf28skd.yaml | sudo /var/lib/rancher/rke2/bin/kubectl --kubeconfig /etc/rancher/rke2/rke2.yaml apply -f -

  7. Instala el Proveedor de Nube Harvester.

    En el panel de control del clúster RKE2, ve a Apps > Charts > Proveedor de Nube Harvester > Instalar.

  8. Instala el Controlador CSI de Harvester.

    En el panel de control del clúster RKE2, ve a Apps > Charts > Controlador CSI de Harvester > Instalar.
