Soporte de registro OCI para Admission Controller

Distribuyes tus directivas de SUSE Security Admission Controller como Artefactos OCI usando registros regulares de la Iniciativa de Contenedores Abiertos (OCI).

El almacenamiento de directivas de Admission Controller está junto a las imágenes de contenedor. No requieren configuración o mantenimiento adicional más allá de lo necesario para las imágenes de contenedor regulares.

Puedes añadir un registro que funcione con Admission Controller o corregir cualquier inexactitud del registro utilizando el botón "Editar esta página" en la parte inferior de la página

Proyectos que implementan registros OCI

  • Harbor.

  • Distribution (≥ 2.7.0).

  • Quay: Soportado, pero inactivo por defecto en v3.6.

Registros OCI alojados

  • GitHub Container Registry. Ver aquí.

  • Quay.io. Ver la lista de proyectos.

  • Amazon ECR: Ver aquí.

  • Google Artifact Registry. Ver aquí.

  • Azure Container Registry. Ver aquí.

  • Docker Hub. Ver aquí.

  • IBM Cloud Container Registry. Ver aquí.

  • JFrog Artifactory. Ver aquí.

Herramientas que funcionan con registros OCI

El proyecto Admission Controller recomienda:

  • kwctl (nuestra herramienta CLI).

  • Skopeo (≥ 1.9.0).

  • Crane.

Problemas conocidos

Docker Hub

Actualmente, Docker Hub no soporta artefactos OCI, por lo que no se puede utilizar para almacenar directivas de Admission Controller. Docker Inc. ha anunciado que Docker Hub soportará artefactos OCI en el futuro.

JFrog

Aunque JFrog soporta artefactos OCI, solo es posible subir a él de forma parcial, siguiendo su especificación. Lee más aquí.
