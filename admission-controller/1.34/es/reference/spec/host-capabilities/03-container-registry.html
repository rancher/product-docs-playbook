Capacidades del registro de contenedores

Los registros de contenedores se pueden utilizar para distribuir muchos tipos de objetos OCI. Desde imágenes de contenedores bien conocidas hasta artefactos OCI genéricos. Los artefactos OCI se utilizan para almacenar objetos como SUSE Security Admission Controller directivas, gráficos de Helm y más.

Estas son las capacidades expuestas por el Admission Controller anfitrión, para interactuar con los registros de contenedores.

Obtener el resumen del manifiesto OCI

Esta devolución de llamada calcula el resumen de un manifiesto OCI. El resumen se puede utilizar para identificar un objeto almacenado en un registro OCI. Este es un método inmutable, a diferencia de tags que son mutables.

Almacenamiento en el caché.

Calcular el resumen implica una serie de solicitudes de red entre el Admission Controller anfitrión de directivas y el registro remoto. Estas operaciones pueden ser costosas en tiempo, por lo que los resultados se almacenan en caché durante 1 minuto.

Autenticación

Las interacciones con registros privados requieren que el Admission Controller anfitrión de directivas se autentique contra el registro remoto.

El anfitrión de directivas utilizará el mismo conjunto de credenciales que se usaron para obtener directivas del registro remoto.

Protocolo de comunicación

Esta es la descripción del protocolo waPC utilizado para exponer esta capacidad:

Función waPC - v1/manifest_digest entrada

# OCI URI - JSON encoded string
string

Función waPC - v1/manifest_digest salida

{
  # digest of the OCI object
  "digest": string
}

Por ejemplo, al solicitar el resumen del manifiesto de la busybox:latest imagen, la carga útil sería:

  • Carga útil de entrada: "busybox:latest"

  • Carga útil de salida: { "digest": "sha256:69e70a79f2d41ab5d637de98c1e0b055206ba40a8145e7bddb55ccc04e13cf8f"}
