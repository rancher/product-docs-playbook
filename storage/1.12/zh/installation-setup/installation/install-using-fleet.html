使用 Fleet 安装 SUSE Storage

先决条件

SUSE Storage 安装需要对 SUSE 应用程序集合的认证访问。有关更多详细信息，请查看 认证指南。

  • 您的工作站：安装 Helm v3.0 或更高版本。

  • Kubernetes 集群：

    • 确保每个节点满足 安装要求。

    • 使用 Helm 安装 SUSE® Rancher Prime: Continuous Delivery。

      helm repo add fleet https://rancher.github.io/fleet-helm-charts/
helm -n cattle-fleet-system install --create-namespace --wait fleet-crd fleet/fleet-crd
helm -n cattle-fleet-system install --create-namespace --wait fleet fleet/fleet

      请稍候片刻，以便在 cattle-fleet-system 的名称空间中部署 Fleet 组件。

可以使用 Longhorn 命令行工具 检查 Longhorn 环境中的潜在问题。

安装

  1. 在您的 GitOps 储存库中，创建一个 fleet.yaml 文件，其中包括以下内容：

    • 在 SUSE Storage 的名称空间中安装 longhorn-system 的参数

       defaultNamespace: longhorn-system

    • 忽略修改的 CRD 的参数

       diff:
   comparePatches:
   - apiVersion: apiextensions.k8s.io/v1
     kind: CustomResourceDefinition
     name: engineimages.longhorn.io
     operations:
     - {"op": "replace", "path": "/status"}
   - apiVersion: apiextensions.k8s.io/v1
     kind: CustomResourceDefinition
     name: nodes.longhorn.io
     operations:
     - {"op": "replace", "path": "/status"}
   - apiVersion: apiextensions.k8s.io/v1

     name: volumes.longhorn.io
     operations:
     - {"op": "replace", "path": "/status"}
   - apiVersion: apiextensions.k8s.io/v1

     name: engines.longhorn.io
     operations:
     - {"op": "replace", "path": "/status"}
   - apiVersion: apiextensions.k8s.io/v1
     kind: CustomResourceDefinition
     name: instancemanagers.longhorn.io
     operations:
     - {"op": "replace", "path": "/status"}
   - apiVersion: apiextensions.k8s.io/v1
     kind: CustomResourceDefinition
     name: replicas.longhorn.io
     operations:
     - {"op": "replace", "path": "/status"}
   - apiVersion: apiextensions.k8s.io/v1
     kind: CustomResourceDefinition
     name: settings.longhorn.io
     operations:
     - {"op": "replace", "path": "/status"}

    • 指定要安装的 Helm 图表版本的参数

     helm:
   repo: https://charts.longhorn.io
   chart: longhorn
   version: v1.12.0 # Replace with the version you would like to install or upgrade to
   releaseName: longhorn

    + 完整 fleet.yaml 文件的示例：

    +

  2. 创建一个指向您的 GitOps 储存库的 GitRepo 自定义资源 (CR)。

     cat > longhorn-gitrepo.yaml << "EOF"
 apiVersion: fleet.cattle.io/v1alpha1
 kind: GitRepo
 metadata:
   name: longhorn
   namespace: fleet-local
 spec:
   repo: https://github.com/your-username/your-gitops-repo.git
   revision: main
   paths:
   - .
 EOF

  3. 应用 GitRepo CR。

     kubectl apply -f longhorn-gitrepo.yaml

  4. 验证 GitRepo CR 是否成功创建并同步。

     kubectl -n fleet-local get gitrepo -w

  5. 验证 SUSE Storage 是否成功安装。

     kubectl -n longhorn-system get pod
