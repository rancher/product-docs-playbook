Métricas do SUSE® Storage para Monitoramento

Volume

Nome Descrição Exemplo

longhorn_volume_actual_size_bytes

Espaço real utilizado por cada réplica do volume no nó correspondente

longhorn_volume_actual_size_bytes{pvc_namespace="default",node="worker-2",pvc="testvol",volume="testvol"} 1.1917312e+08

longhorn_volume_capacity_bytes

Tamanho configurado em bytes para este volume

longhorn_volume_capacity_bytes{pvc_namespace="default",node="worker-2",pvc="testvol",volume="testvol"} 6.442450944e+09

longhorn_volume_state

Estado do volume. Esta métrica usa o rótulo state para indicar o estado atual do volume. O valor é 1 para o estado atual e 0 para os outros. Estados: criando, anexado, desanexado, anexando, desanexando, deletando

longhorn_volume_state{pvc_namespace="default",node="worker-2",pvc="testvol",volume="testvol",state="attached"} 1

longhorn_volume_robustness

Robustez do volume. Esta métrica usa o rótulo state para indicar a robustez atual. O valor é 1 para o estado atual e 0 para os outros. Estados: desconhecido, saudável, degradado, com falha

longhorn_volume_robustness{pvc_namespace="default",node="worker-2",pvc="testvol",volume="testvol",state="healthy"} 1

longhorn_volume_write_iops

Esta métrica indica que o volume agora está em modo somente leitura. A métrica é 1 ou nenhum registro para cada volume

CSI

O componente sidecar CSI possui métricas incorporadas para que os usuários obtenham insights sobre as operações do CSI. As métricas das operações do CSI cobrem contagem total, contagem de erros e latência de chamadas. O Longhorn habilita as métricas adicionando a flag --http-endpoint para cada componente sidecar do CSI. Você pode usar PodMonitor do Prometheus para coletar essas métricas.

Nome Portar

longhorn-csi-attacher

8000

longhorn-csi-provisioner

8000

longhorn-csi-resizer

8000

longhorn-csi-snapshotter

8000
