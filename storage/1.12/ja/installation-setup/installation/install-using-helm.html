Helmを使用してSUSE Storageをインストールします。

前提条件

SUSE Storageのインストールには、SUSE Application Collectionへの認証されたアクセスが必要です。詳細については、 認証ガイドを確認してください。

[ツール]

  • Helmのバージョン3.0以降が必要です。

  • Longhornコマンドラインツールを使用して、SUSE Storage環境の潜在的な問題を確認できます。

Kubernetes環境

  • 各ノードがインストール要件を満たすKubernetesクラスターが必要です。

認証

  • SUSE Application Collection OCIチャートリポジトリにHelmを認証するには、 Helm認証ドキュメントを参照してください。

  • Kubernetesがレジストリからイメージをプルできるようにするには、 Kubernetes認証ドキュメントに従ってシークレットを作成し、`global.imagePullSecrets`チャート値を設定してください。

インストール

  1. デフォルト設定でSUSE Storageをインストールするには、次のコマンドを実行します：

    helm install longhorn oci://dp.apps.rancher.io/charts/suse-storage \
  --namespace longhorn-system \
  --create-namespace \
  --version <version> \ # Replace with the version you would like to install
  --set global.imagePullSecrets=<PULL_IMAGE_SECRET>

  2. デプロイメントが成功したことを確認するには、次のコマンドを実行します：

    kubectl -n longhorn-system get pod

  4. システムアクセス手順を使用してUIにアクセスします。

設定

SUSE Storage Helmチャートは、特定のニーズに応じた広範な設定オプションを提供します。これらのオプションは、values.yaml`ファイルを提供するか、--set`フラグを使用して`helm install`または`helm upgrade`の際に設定できます。

一般的に設定可能な側面には以下が含まれます：

  • ストレージクラス:デフォルトのStorageClass設定です。

  • UI設定:SUSE Storage UIのためのIngress構成、サービスタイプ、および認証オプション。

  • マネージャーとエンジンの設定:SUSE Storage コアコンポーネントのリソース制限、レプリカ数、およびその他の運用パラメータ。

  • ノードとディスクの管理:自動ディスクプロビジョニング、ノードセレクター、およびトレランス。

  • CSIドライバー:コンテナストレージインターフェースコンポーネントに特有の設定。

  • ネットワーキング:サービスタイプとネットワークポリシー。

すべての利用可能なパラメータとその説明の包括的なリストについては、公式のSUSE Storage Helm Chartドキュメントおよびチャートのリポジトリ内の`values.yaml`ファイルを参照してください。
