Configuraciones por defecto del sistema

Durante la instalación, puedes permitir que SUSE Storage utilice las configuraciones por defecto del sistema o usar banderas específicas para modificar los valores por defecto. Después de la instalación, puedes modificar las configuraciones utilizando la interfaz de usuario de SUSE Storage. Para más información, consulta la sección Referencia de Configuración de la documentación.

Configuración que permite a SUSE Storage recopilar periódicamente datos de uso anónimos con fines de mejora del producto. SUSE Storage envía los datos recopilados al servidor Upgrade Responder, que es la fuente de datos del Panel de Métricas Públicas de Longhorn (https://metrics.longhorn.io). El servidor Upgrade Responder no almacena datos que puedan ser utilizados para identificar a los clientes, incluyendo direcciones IP.

defaultSettings.allowEmptyDiskSelectorVolume

Configuración que permite la programación de volúmenes de selector de disco vacíos a cualquier disco.

Configuración que permite la programación de volúmenes de selector de nodo vacíos a cualquier nodo.

defaultSettings.allowRecurringJobWhileVolumeDetached

Configuración que permite a SUSE Storage adjuntar automáticamente un volumen y crear instantáneas o copias de seguridad cuando se ejecutan trabajos recurrentes.

defaultSettings.allowVolumeCreationWithDegradedAvailability

Configuración que te permite crear y adjuntar un volumen sin tener todas las réplicas programadas en el momento de la creación.

defaultSettings.autoCleanupRecurringJobBackupSnapshot

Configuración que permite a SUSE Storage eliminar automáticamente un pod de carga de trabajo que es gestionado por un controlador (por ejemplo, daemonset) cada vez que un volumen de Longhorn se desacopla inesperadamente (por ejemplo, durante las actualizaciones de Kubernetes). defaultSettings.autoSalvage

Configuración que permite a SUSE Storage recuperar automáticamente volúmenes cuando todas las réplicas se vuelven defectuosas (por ejemplo, cuando se interrumpe la conexión de red). defaultSettings.backingImageCleanupWaitInterval

Número de minutos que SUSE Storage espera antes de limpiar el archivo de imagen de respaldo cuando no hay réplicas en el disco utilizándolo.

defaultSettings.backingImageRecoveryWaitInterval

Número de segundos que SUSE Storage espera antes de descargar de nuevo un archivo de imagen de respaldo cuando el estado de todos los archivos de imagen de disco cambia a "fallido" o "desconocido".

defaultSettings.backupCompressionMethod

defaultSettings.backupConcurrentLimit

Número máximo de hilos de trabajo que pueden ejecutarse de forma concurrente para cada copia de seguridad.

Número de minutos que Longhorn permite para la ejecución de la copia de seguridad. El valor por defecto es "1".

defaultSettings.blacklistForAutoDeletePodWhenVolumeDetachedUnexpectedly

Lista negra de valores de api/kind de controlador para la configuración Eliminar automáticamente el Pod de carga de trabajo cuando el Volumen se desconecta inesperadamente. Si un pod de carga de trabajo es gestionado por un controlador cuyo api/kind está listado en esta lista negra, Longhorn no eliminará automáticamente el pod cuando su volumen se desconecte inesperadamente. Se pueden especificar múltiples entradas de api/kind de controlador, separadas por punto y coma. Por ejemplo: apps/StatefulSet;apps/DaemonSet. Nota que el api/kind del controlador es sensible a mayúsculas y debe coincidir exactamente con el api/kind en la referencia del propietario del pod de carga de trabajo.

Número máximo de motores que se permiten actualizar de forma concurrente en cada nodo después de que se actualiza Longhorn Manager. Cuando el valor es "0", SUSE Storage no actualiza automáticamente los motores de volumen a la nueva versión de imagen de motor por defecto.

Número máximo de réplicas que se pueden reconstruir de forma concurrente en cada nodo.

Número máximo de volúmenes que se pueden restaurar de forma concurrente en cada nodo utilizando una copia de seguridad. Cuando el valor es "0", la restauración de volúmenes utilizando una copia de seguridad está deshabilitada.

defaultSettings.dataEngineLogLevel

Aplica solo al Motor de Datos V2. Especifica el nivel de registro para el demonio objetivo del Storage Performance Development Kit (SPDK). Los valores admitidos son: Error, Warning, Notice, Info y Debug. El valor por defecto es Notice.

defaultSettings.dataEngineMemorySize

Aplica solo al Motor de Datos V2. Especifica el tamaño de hugepage, en MiB, para el demonio objetivo del Storage Performance Development Kit (SPDK). El valor por defecto es "{"v2":"2048"}".

Especifica el tamaño de bloque de copia de seguridad por defecto, en MiB, utilizado al crear un nuevo volumen. Los valores admitidos son 2 o 16.

defaultSettings.defaultDataLocality

Localidad de datos por defecto. Un volumen de Longhorn tiene localidad de datos si existe una réplica local del volumen en el mismo nodo que el pod que está utilizando el volumen.

defaultSettings.defaultDataPath

Ruta por defecto para almacenar datos en un host. Una ruta de directorio absoluta indica un disco de tipo sistema de archivos utilizado por el V1 Data Engine, mientras que una ruta a un dispositivo de bloque indica un disco de tipo bloque utilizado por el V2 Data Engine. defaultSettings.defaultReplicaCount

Número por defecto de réplicas para volúmenes creados utilizando la interfaz de usuario SUSE Storage. Para la configuración de Kubernetes, modifica el campo numberOfReplicas en la clase de almacenamiento. El valor por defecto es "{"v1":"3","v2":"3"}".

defaultSettings.deletingConfirmationFlag

Configuración que permite la separación automática de volúmenes adjuntos manualmente cuando un nodo está aislado.

defaultSettings.engineReplicaTimeout

Tiempo de espera entre el motor de Longhorn y las réplicas. Especifica un valor entre "8" y "30" segundos. El valor por defecto es "8".

defaultSettings.failedBackupTTL

Configuración que congela el sistema de archivos en la partición root antes de que se cree una instantánea.

defaultSettings.logLevel

Habilita la reconstrucción automática de réplicas degradadas mientras el volumen está desconectado. Esta configuración solo tiene efecto si la configuración del volumen individual está establecida en "ignored" o "enabled".

Porcentaje de capacidad mínima de disco disponible. Cuando la capacidad mínima disponible excede la capacidad total disponible, el disco se vuelve no programable hasta que se libere más espacio para su uso. El valor por defecto es "25".

Ten en cuenta que cambiar los límites de recursos provocará el reinicio de los componentes de CSI, lo que puede afectar temporalmente el aprovisionamiento de volúmenes y las operaciones de adjuntar/desadjuntar hasta que los componentes estén listos. Selector de nodo para componentes de Longhorn gestionados por el sistema.

defaultSettings.systemManagedPodsImagePullPolicy

Política de extracción de imágenes para pods gestionados por el sistema, como Instance Manager, imágenes de motor y CSI Driver. Los cambios en la política de extracción de imágenes se aplican solo después de que los pods gestionados por el sistema se reinicien.

defaultSettings.taintToleration

Taint o tolerancia para componentes de Longhorn gestionados por el sistema. Especifica los valores utilizando una lista separada por punto y coma en la sintaxis kubectl taint (Ejemplo: key1=value1:effect; key2=value2:effect).

defaultSettings.upgradeChecker

Comprobador de actualización que verifica periódicamente nuevas versiones de SUSE Storage. Cuando hay una nueva versión disponible, aparece una notificación en la interfaz de usuario de SUSE Storage. Este ajuste está habilitado por defecto.

defaultSettings.upgradeResponderURL

El Respondedor de actualización envía una notificación cada vez que hay una nueva versión de Longhorn a la que puedes actualizar. El valor por defecto es https://longhorn-upgrade-responder.rancher.io/v1/checkupgrade.

defaultSettings.v1DataEngine

Configuración que te permite habilitar el Motor de Datos V1.

defaultSettings.v2DataEngine

Configuración que te permite habilitar el Motor de Datos V2, que se basa en el Kit de Desarrollo de Rendimiento de Almacenamiento (SPDK). El Motor de Datos V2 es una característica experimental y no debe utilizarse en entornos de producción.
