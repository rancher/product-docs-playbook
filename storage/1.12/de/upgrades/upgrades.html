Durchsetzung des Upgrade-Pfades und Verhinderung von Downgrades

Ab Version SUSE Storage v1.5.0 werden Upgrades nur von einer Minor-Version zur nächsten aufeinanderfolgenden Minor-Version unterstützt. Zum Beispiel können Sie von 1.5.x auf 1.6.x upgraden, aber das Überspringen von Versionen (wie von 1.4.x auf 1.6.x) wird nicht unterstützt. Wenn Sie versuchen, von einer nicht unterstützten Version ein Upgrade durchzuführen oder eine Minor-Version zu überspringen, schlägt die Operation automatisch fehl. Sie können jedoch zur zuvor installierten Version ohne Unterbrechung des Dienstes oder Ausfallzeit zurückkehren.

Darüber hinaus unterstützt SUSE Storage keine Downgrades. Diese Einschränkung hilft, unerwartetes Systemverhalten und Probleme im Zusammenhang mit Funktionsinkompatibilität, Auslaufen oder Entfernung zu verhindern.

  • Sobald Sie erfolgreich auf v1.12.0 ein Upgrade durchgeführt haben, dürfen Sie nicht zur zuvor installierten Version zurückkehren.

  • Downgrades werden nicht unterstützt und sind daher nicht empfohlen.

Manuelle Überprüfungen vor Upgrades

Automatisierte Überprüfungen werden nur für einige Upgrade-Pfade durchgeführt, und der Vor-Upgrade-Checker deckt möglicherweise nicht alle Szenarien ab. Manuelle Überprüfungen, die entweder mit kubectl oder der Benutzeroberfläche durchgeführt werden, werden für diese Szenarien empfohlen. Sie können mildernde Maßnahmen ergreifen oder das Upgrade aufschieben, bis die Probleme behoben sind.

  • Stellen Sie sicher, dass alle V2 Data Engine-Volumes getrennt und die Replikate gestoppt sind. Die V2 Data Engine unterstützt derzeit keine Live-Upgrades.

  • Vermeiden Sie Upgrades, wenn Volumes den Status "Fehlerhaft" haben. Wenn alle Replikate als unbrauchbar erachtet werden, können sie gelöscht werden und Daten können dauerhaft verloren gehen (wenn keine brauchbaren Sicherungen vorhanden sind).

  • Vermeiden Sie Upgrades, wenn ein fehlgeschlagenes Backing Image vorhanden ist. Für weitere Informationen siehe Backing Image.

  • Erstellen Sie ein Longhorn-System-Backup, bevor Sie das Upgrade durchführen. Siehe Longhorn-System-Backup für Anweisungen. Ein System-Backup stellt sicher, dass alle kritischen Ressourcen, wie Volumes und Backing Images, gesichert sind und wiederhergestellt werden können, falls Probleme auftreten.

Upgrade SUSE Storage

Normalerweise gibt es zwei Schritte im Upgrade-Prozess: Zuerst den Longhorn Manager auf die neueste Version upgraden, dann die Longhorn Engine manuell auf die neueste Version mit dem neuesten Longhorn Manager upgraden.

1. Upgrade Longhorn Manager

  • Um von v1.12.x zu upgraden, Upgrade Longhorn Manager.

2. Manuelles Upgrade der Longhorn Engine

Nachdem der Longhorn Manager upgegradet wurde, muss auch die Longhorn Engine upgegradet werden unter Verwendung der SUSE Storage Benutzeroberfläche.

3. Automatisches Upgrade der Longhorn Engine

Seit SUSE Storage v1.1.1 bieten wir eine Option an, um Ihnen zu helfen, Engines automatisch zu upgraden.

4. Automatische Migration wiederkehrender Jobs

Mit der Einführung des neuen, labelgesteuerten Recurring Job Features hat SUSE Storage das RecurringJobs Feld in der Volume-Spezifikation entfernt und plant, RecurringJobs in der StorageClass auslaufen zu lassen.

Während des Upgrades führt SUSE Storage automatisch Folgendes aus:

  • Erstellt neue CRs für wiederkehrende Jobs aus dem recurringJobs Feld in der Volume-Spezifikation und konvertiert sie in die Volume-Labels.

  • Erstellt neue CRs für wiederkehrende Jobs aus dem recurringJobs in der StorageClass und konvertiert sie in den neuen recurringJobSelector Parameter.

Besuchen Sie Wiederkehrende Snapshot und Sicherungen für weitere Informationen über das neue Recurring Job Feature.
