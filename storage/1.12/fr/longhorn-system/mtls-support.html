Support TLS mutuel (mTLS)

Longhorn prend en charge mTLS pour sécuriser et chiffrer la communication grpc entre le plan de contrôle (longhorn-manager) et le plan de données (instance-managers). Pour la configuration du certificat, nous utilisons le mécanisme de secret Kubernetes en combinaison avec un montage de secret optionnel pour le longhorn-manager/instance-manager.

Configuration requise

Dans une installation par défaut, mTLS est désactivé ; pour activer le support mTLS, il faut créer un secret longhorn-grpc-tls dans l’espace de noms longhorn-system avant le déploiement. Le secret est spécifié comme un montage de secret optionnel pour le longhorn-manager/instance-managers, donc s’il n’existe pas lorsque ces composants sont démarrés, mTLS ne sera pas utilisé et un redémarrage des composants sera nécessaire pour activer le support TLS.

Le longhorn-manager a un client non TLS de secours pour les configurations en mode mixte où il y a d’anciens instance-managers qui ont été démarrés sans support TLS.

Configuration du certificat auto-signé

Vous devez créer un ca.crt avec le drapeau CA activé, qui sera ensuite utilisé pour signer le tls.crt, ce qui vous permettra de faire pivoter le tls.crt à l’avenir sans interruption de service. Vous pouvez utiliser openssl ou cfssl pour le ca.crt ainsi que pour la génération de certificats tls.crt.

Le certificat tls.crt doit utiliser longhorn-backend pour le nom commun et la liste ci-dessous d’entrées pour le Nom Alternatif du Sujet.

Common Name: longhorn-backend
Subject Alternative Names: longhorn-backend, longhorn-backend.longhorn-system, longhorn-backend.longhorn-system.svc, longhorn-frontend, longhorn-frontend.longhorn-system, longhorn-frontend.longhorn-system.svc, longhorn-engine-manager, longhorn-engine-manager.longhorn-system, longhorn-engine-manager.longhorn-system.svc, longhorn-replica-manager, longhorn-replica-manager.longhorn-system, longhorn-replica-manager.longhorn-system.svc, longhorn-csi, longhorn-csi.longhorn-system, longhorn-csi.longhorn-system.svc, longhorn-backend, IP Address:127.0.0.1

Configuration des Secrets Kubernetes

Le ca.crt est le certificat de l’autorité de certification qui a été utilisé pour signer le tls.crt, qui sera utilisé à la fois par le client (longhorn-manager) et le serveur (instance-manager) pour l’authentification grpc mTLS. La tls.key est la clé privée associée au tls.crt créé.
