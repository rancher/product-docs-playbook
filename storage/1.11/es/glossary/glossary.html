Adjuntar/Readjuntar

Adjuntar un dispositivo de bloque es hacerlo aparecer en el nodo de Linux (por ejemplo, /dev/longhorn/testvol). Si el motor de volúmenes falla inesperadamente, SUSE Storage readjunta automáticamente el volumen.

Copia de seguridad

Una copia de seguridad es un objeto almacenado en el backupstore. El backupstore puede contener tanto copias de seguridad de volúmenes como copias de seguridad del sistema.

Backupstore

El backupstore es la ubicación de almacenamiento externo donde se guardan las copias de seguridad de SUSE Storage. Puede ser un recurso compartido NFS, un recurso compartido CIFS o un almacén de objetos compatible con S3. SUSE Storage se conecta a un backupstore a través de un objetivo de copia de seguridad configurado.

Objetivo de Copia de Seguridad

Un objetivo de copia de seguridad es el punto final utilizado para acceder a un backupstore en SUSE Storage.

Volumen de Copia de Seguridad

Un volumen de copia de seguridad representa todas las copias de seguridad asociadas con un único volumen original de Longhorn. Se almacena en el backupstore y es visible en la página de Copia de seguridad de la interfaz de usuario de SUSE Storage.

Los volúmenes de copia de seguridad contienen múltiples copias de seguridad para el mismo volumen. Las copias de seguridad se crean a partir de instantáneas y capturan el estado del volumen en el momento en que se tomó la instantánea. No incluyen la cadena de instantáneas ni el historial de cambios.

Las copias de seguridad se almacenan como bloques de 2 MiB en el almacenamiento de objetos por defecto.

Para más detalles sobre cómo funcionan las instantáneas y las copias de seguridad, consulta la documentación de conceptos - Cómo Funcionan las Copias de Seguridad.

Almacenamiento en bloques

Un enfoque de almacenamiento en el que los datos se almacenan en bloques de tamaño fijo, cada uno identificado por una dirección de memoria.

Discos de tipo bloque

Un disco de tipo bloque es un dispositivo de bloque sin formatear que SUSE Storage utiliza directamente para volúmenes del motor de datos V2.

CRD

Una definición de recurso personalizado de Kubernetes.

Recuperación ante desastres entre clústeres

La recuperación ante desastres entre clústeres permite recuperar datos de un clúster primario de Kubernetes en un clúster separado utilizando copias de seguridad.

Controlador CSI

El controlador CSI de Longhorn es una interfaz de almacenamiento de contenedores que se puede utilizar con Kubernetes. El controlador CSI para volúmenes de Longhorn se llama driver.longhorn.io.

Volúmenes de recuperación ante desastres (volumen DR)

Un volumen DR es un volumen especial utilizado para mantener una copia de los datos en un clúster de copia de seguridad para que la carga de trabajo pueda recuperarse si el clúster primario no está disponible. Los volúmenes DR se utilizan para aumentar la resiliencia de los volúmenes de Longhorn.

Cada volumen de copia de seguridad en el backupstore corresponde a un volumen original, y cada volumen DR corresponde a un volumen de copia de seguridad.

Los volúmenes DR se pueden crear para reflejar con precisión las copias de seguridad de un volumen de Longhorn, pero no pueden funcionar como volúmenes normales de Longhorn hasta que se activen.

Ext4

Un sistema de archivos de Linux soportado por SUSE Storage para almacenamiento.

Discos de tipo sistema de archivos

Un disco de tipo sistema de archivos es un dispositivo de bloque formateado con un sistema de archivos basado en extensiones (como ext4 o XFS). SUSE Storage lo monta y almacena datos de réplica del motor de datos V1.

Frontend

El frontend se refiere a un dispositivo de bloque expuesto por un volumen de Longhorn.

Instance Manager

El componente SUSE Storage responsable de gestionar el ciclo de vida de las instancias de motor y réplica.

Longhorn Engine

Un motor de Longhorn es un componente de plano de datos de SUSE Storage. Es un controlador de almacenamiento por volumen responsable de replicar datos de forma sincrónica a las réplicas del volumen.

Longhorn Manager

El Longhorn Manager es el componente de plano de control de SUSE Storage. Se ejecuta como un DaemonSet de Kubernetes y es responsable de gestionar volúmenes, manejar solicitudes de API y orquestar motores de Longhorn, réplicas y otros recursos personalizados.

Longhorn Volume

Un volumen de Longhorn es un volumen de Kubernetes gestionado y replicado por SUSE Storage.

Para cada volumen, el Longhorn Manager crea:

  • una instancia de Longhorn Engine

  • múltiples réplicas, cada una conteniendo una cadena de instantáneas que representa el historial del volumen

Por defecto, se crean tres réplicas y se distribuyen entre diferentes nodos para garantizar alta disponibilidad.

Modo de mantenimiento

Un modo de adjunto de volumen que adjunta el volumen sin habilitar el frontend, utilizado principalmente para revertir un volumen a partir de una instantánea.

Montar

Un comando de Linux utilizado para adjuntar un dispositivo de bloque a un directorio en el nodo (por ejemplo, mount /dev/longhorn/testvol/mnt).

NFS

Un protocolo de sistema de archivos distribuido que proporciona acceso a archivos basado en red.

SUSE Storage admite el uso de NFS en los siguientes casos:

  • Como backupstore para almacenamiento secundario.

  • Como el sistema de archivos subyacente para volúmenes RWX.

Almacenamiento de objetos

Una arquitectura de almacenamiento de datos que gestiona los datos como objetos, cada uno conteniendo los datos, una cantidad variable de metadatos y un identificador único global. SUSE Storage admite la copia de seguridad de volúmenes en almacenes de objetos compatibles con S3.

Expansión fuera de línea

Una expansión de volumen realizada mientras el volumen está desconectado.

Sobreaprovisionamiento

El sobreaprovisionamiento permite asignar más almacenamiento lógico del que la capacidad física disponible.

Por ejemplo, un nodo con 200 GiB de espacio en disco (con 50 GiB reservados para el sistema operativo) podría aprovisionar 750 GiB de volúmenes Longhorn.

El Porcentaje de Sobreaprovisionamiento de Almacenamiento es configurable en la configuración de SUSE Storage. Consulta Configuración para más detalles.

PersistentVolume

Un PersistentVolume (PV) es un recurso de Kubernetes que representa una parte del almacenamiento en el clúster. Puede ser aprovisionado manualmente o dinámicamente utilizando StorageClasses.

Es un recurso a nivel de clúster y es necesario para que los pods utilicen almacenamiento persistente que es independiente del ciclo de vida del pod.

Para más información, consulta la documentación de Kubernetes sobre volúmenes persistentes.

PersistentVolumeClaim

Un PersistentVolumeClaim (PVC) es una solicitud de usuario para almacenamiento. Las reclamaciones especifican el tamaño deseado y los modos de acceso (por ejemplo, ReadWriteOnce o ReadOnlyMany).

Los pods utilizan PVCs para obtener almacenamiento persistente.

Consulta la documentación oficial de Kubernetes en volúmenes persistentes.

Copias de seguridad primarias

Las réplicas de un volumen Longhorn dentro del clúster de Kubernetes pueden considerarse copias de seguridad primarias.

Leer índice

El índice de lectura es una estructura de datos en memoria utilizada por cada réplica para mejorar el rendimiento de lectura. Registra qué disco de diferencias (instantánea) contiene los datos válidos para cada bloque de 4K.

Instantáneas recurrentes

Las instantáneas recurrentes permiten que SUSE Storage cree y retenga automáticamente instantáneas a una frecuencia especificada (por ejemplo, cada hora o diariamente).

Volver a montar

Después de la reconexión, SUSE Storage detecta y monta automáticamente el sistema de archivos del volumen.

Réplica

Una réplica es un almacén de datos de volumen Longhorn que consiste en un historial de los cambios en los datos dentro de un volumen.

S3

Amazon S3 es un servicio de almacenamiento de objetos.

Recuperar un volumen

Se requiere una operación de recuperación cuando todas las réplicas se vuelven defectuosas (por ejemplo, debido a una desconexión de red). Durante la recuperación, SUSE Storage intenta identificar cualquier réplica utilizable y luego las utiliza para recuperar el volumen.

Copias de seguridad secundarias

Copias de seguridad almacenadas externamente al clúster de Kubernetes, en S3, NFS o CIFS.

SMB/CIFS

Un protocolo de compartición de archivos en red que proporciona acceso remoto a archivos similar al almacenamiento local. SUSE Storage admite el uso de SMB/CIFS como repositorio de copias de seguridad para almacenamiento secundario.

Instantánea

Una instantánea captura el estado de un volumen en el momento en que se crea la instantánea. Cada instantánea almacena solo los cambios que sobrescriben datos anteriores, por lo que se requiere una cadena de instantáneas para representar el estado completo.

Los volúmenes se pueden restaurar a partir de instantáneas.

Consulta la documentación de conceptos - Cómo funcionan las instantáneas.

Integridad de los datos de la instantánea

La integridad de los datos de la instantánea es una característica SUSE Storage que calcula el hash de los archivos de disco de la instantánea y verifica periódicamente su integridad para detectar corrupciones que el sistema de archivos no detecta, como la degradación de bits.

Identidad estable

StatefulSets tienen una identidad estable, lo que significa que Kubernetes no elimina forzosamente el Pod.

StatefulSet

Un recurso de Kubernetes utilizado para gestionar aplicaciones con estado.

StorageClass

Un recurso de Kubernetes utilizado para aprovisionar automáticamente PersistentVolumes para pods.

Copia de seguridad del sistema

Una copia de seguridad del sistema contiene un paquete de recursos del sistema Longhorn y se almacena en el almacenamiento de copias de seguridad.

Consulta la documentación del paquete de copia de seguridad del sistema Longhorn para más detalles sobre los recursos incluidos en una copia de seguridad del sistema.

Aprovisionamiento flexible

SUSE Storage es un sistema de almacenamiento con aprovisionamiento fino. El volumen SUSE Storage utiliza únicamente el espacio requerido en cada momento.

Por ejemplo, un volumen de 20 GiB que almacena 1 GiB de datos utiliza solo 1 GiB de espacio en disco.

Umount

Un comando de Linux que desacopla un sistema de archivos de la jerarquía de archivos.

Motor de datos V1

El motor de datos V1 es el componente original del plano de datos de SUSE Storage. Utiliza archivos dispersos del sistema de archivos del host para un aprovisionamiento fino, con volúmenes compuestos de réplicas, que admiten instantáneas y copias de seguridad.

Motor de datos V2

El motor de datos V2 es el plano de datos de Longhorn. Utiliza SPDK y páginas enormes, y dispositivos de bloque para proporcionar un rendimiento superior.

Volúmenes (Concepto de Kubernetes)

Un volumen de Kubernetes permite a un pod almacenar archivos durante su vida útil. Estos archivos persisten a través de reinicios de contenedor, pero no cuando se elimina el pod.

Para preservar el almacenamiento más allá del ciclo de vida del pod, se requiere un PersistentVolume.

Consulta la documentación de Kubernetes en Volumes para más detalles.

XFS

Un sistema de archivos compatible con la mayoría de las distribuciones de Linux. SUSE Storage admite XFS como almacenamiento.
