Rancher Clúster de Windows

Rancher puede aprovisionar un clúster de Windows con una combinación de nodos de trabajo Linux y nodos de trabajo Windows. Para más información sobre el clúster de Windows de Rancher, consulta la documentación oficial de Rancher.

En un clúster de Windows de Rancher, todos los nodos de trabajo Linux están:

  • Marcados con el taint cattle.io/os=linux:NoSchedule

  • Etiquetados con kubernetes.io/os:linux

Después de que se despliegue SUSE Storage, puedes lanzar cargas de trabajo que utilicen volúmenes SUSE Storage solo en nodos Linux.

Desplegar SUSE Storage con el gráfico de Helm soportado

Puedes actualizar el valor de Helm global.cattle.windowsCluster.enabled para permitir la instalación de SUSE Storage en el clúster de Windows de Rancher.

Cuando este valor se establece en true, SUSE Storage reconocerá el clúster de Windows de Rancher y luego desplegará los componentes SUSE Storage con el selector de nodos y las tolerations correctas para que todas las cargas de trabajo SUSE Storage puedan lanzarse solo en nodos Linux.

En el mercado de Rancher, la configuración se puede personalizar en customize Helm options antes de la instalación:
Edit Options > Other Settings > Rancher Windows Cluster

También en:
Edit YAML

global:
  cattle:
    systemDefaultRegistry: ""
    windowsCluster:
      # Enable this to allow Longhorn to run on the Rancher deployed Windows cluster
      enabled: true

Configurar SUSE Storage componentes para el despliegue existente de SUSE Storage

Puedes configurar el existente SUSE Storage cuando no está desplegado con el gráfico de Helm soportado.

  • Dado que los SUSE Storage componentes solo pueden ejecutarse en nodos Linux, necesitas establecer el selector de nodos kubernetes.io/os:linux para SUSE Storage para seleccionar los nodos Linux. Para más información, consulta Selector de Nodos.

  • Dado que todos los nodos de trabajo Linux en un clúster de Windows de Rancher tienen el taint cattle.io/os=linux:NoSchedule, debes establecer la toleration cattle.io/os=linux:NoSchedule para permitir que SUSE Storage se ejecute en esos nodos. Para más información, consulta Taint Toleration.

SUSE Storage se puede desplegar de manera efectiva en clústeres que comprenden tanto nodos de Windows como de Linux, ofreciendo una solución versátil para diversas necesidades de infraestructura. Para facilitar la instalación desde Rancher, los usuarios deben primero añadir el registro OCI de AppCo como un repositorio de aplicaciones de Rancher. Las instrucciones detalladas para este proceso se pueden encontrar en Documentación de Rancher: Integrar con Rancher Manager
