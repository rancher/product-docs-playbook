Soporte de TLS mutuo (mTLS)

Longhorn soporta MTLS para asegurar y cifrar la comunicación grpc entre el plano de control (longhorn-manager) y el plano de datos (instance-managers). Para la configuración del certificado, utilizamos el mecanismo de secreto de Kubernetes en combinación con un montaje de secreto opcional para el longhorn-manager/instance-manager.

Requisitos

En una instalación por defecto, mTLS está deshabilitado; para habilitar el soporte de mTLS, es necesario crear un secreto longhorn-grpc-tls en el espacio de nombres longhorn-system antes de la ampliación. El secreto se especifica como un montaje de secreto opcional para el longhorn-manager/instance-managers, por lo que si no existe cuando se inician estos componentes, no se utilizará mTLS y será necesario reiniciar los componentes para habilitar el soporte de TLS.

El longhorn-manager tiene un cliente sin TLS como respaldo para configuraciones de modo mixto donde hay instance-managers antiguos que se iniciaron sin soporte de TLS.

Configuración de Certificado Autofirmado

Deberías crear un ca.crt con la bandera CA activada, que luego se utilizará para firmar el tls.crt; esto te permitirá rotar el tls.crt en el futuro sin interrupciones en el servicio. Puedes usar openssl o cfssl para el ca.crt, así como para la generación de certificados tls.crt.

El certificado tls.crt debería usar longhorn-backend para el nombre común y la lista de entradas a continuación para el Nombre Alternativo del Sujeto.

Common Name: longhorn-backend
Subject Alternative Names: longhorn-backend, longhorn-backend.longhorn-system, longhorn-backend.longhorn-system.svc, longhorn-frontend, longhorn-frontend.longhorn-system, longhorn-frontend.longhorn-system.svc, longhorn-engine-manager, longhorn-engine-manager.longhorn-system, longhorn-engine-manager.longhorn-system.svc, longhorn-replica-manager, longhorn-replica-manager.longhorn-system, longhorn-replica-manager.longhorn-system.svc, longhorn-csi, longhorn-csi.longhorn-system, longhorn-csi.longhorn-system.svc, longhorn-backend, IP Address:127.0.0.1

Configurando Secretos de Kubernetes

El ca.crt es el certificado de la autoridad certificadora que se utilizó para firmar el tls.crt, que será utilizado tanto por el cliente (longhorn-manager) como por el servidor (instance-manager) para la autenticación gRPC mTLS. La tls.key es la clave privada asociada al tls.crt creado.
