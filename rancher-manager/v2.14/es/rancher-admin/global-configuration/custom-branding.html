Marca Personalizada

Requisitos previos

Necesitarás tener al menos permisos de miembro del clúster.

Configuración de Marca

Para configurar los ajustes de la marca,

  1. Haz clic en ☰ > Configuración Global.

  2. Haz clic en Branding.

Nombre de la Empresa de Marca Privada

Esta opción reemplaza "Rancher" con el valor que proporciones en la mayoría de los lugares. Los archivos que necesitan tener Rancher en el nombre, como rancher-compose.yml, no se cambiarán.

Enlaces de Soporte

Utiliza una dirección URL para enviar nuevos informes de "Reportar un Problema" en lugar de enviar a los usuarios a la página de problemas de GitHub. Opcionalmente, muestra enlaces de soporte de la comunidad de Rancher.

Logo

Sube logotipos claros y oscuros para reemplazar el logotipo de Rancher en el encabezado de navegación de nivel superior.

Color Primario

Puedes anular el color primario utilizado en toda la interfaz de usuario con un color personalizado de tu elección.

Banners Fijos

Muestra un banner fijo personalizado en el encabezado, pie de página, o en ambos.

Para configurar los ajustes del banner,

  1. Haz clic en ☰ > Configuración Global.

  2. Haz clic en Banners.

Enlaces de navegación personalizados

En esta sección, aprenderás a configurar los enlaces en la barra de navegación izquierda del Tablero del Clúster. Para llegar al tablero del clúster,

  1. En la esquina superior izquierda, haz clic en ☰ > Gestión de Clústeres.

  2. En la página Clústeres, ve al clúster donde deseas enlaces de navegación personalizados y haz clic en Explorar.

Puede ser útil añadir un enlace para un acceso rápido a los servicios instalados en un clúster. Por ejemplo, podrías añadir un enlace a la interfaz de Kiali para clústeres con Istio instalado, o podrías añadir un enlace a la interfaz de Grafana para clústeres con monitoreo de Rancher instalado.

Los enlaces personalizados no afectan quién tiene acceso a cada servicio.

Los enlaces se pueden crear a nivel superior y varios enlaces se pueden agrupar.

Añadiendo un enlace de navegación personalizado

Requisito previo:

Necesitarás tener al menos permisos de miembro del clúster o del proyecto.

  1. Haz clic en ☰ > Gestión de Clústeres.

  2. En la página Clústeres, ve al clúster donde te gustaría añadir enlaces de navegación personalizados y haz clic en Explorar.

  3. En el menú de navegación superior, haz clic en 🔍 (Búsqueda de Recursos).

  4. Escribe Nav y haz clic en Enlaces de Nav.

  5. Haz clic en Crear desde YAML.

  6. La forma más sencilla de crear un enlace de navegación es añadir estos campos:

    name: linkname
toURL: https://example.com

    For more details on setting up links, including optional fields, see Link Configuration.

  7. Haga clic en Crear.

name

annotation

Una anotación de Kubernetes para el recurso personalizado NavLink.

label

Una etiqueta de Kubernetes para el recurso personalizado NavLink.

sideLabel

Etiqueta que aparece en la barra de navegación izquierda

target

Establece la propiedad target de la etiqueta de anclaje del enlace (<a>), que (dependiendo de los navegadores) determina si se abre en una nueva ventana o en una pestaña existente.

El valor predeterminado es _self, que abre el enlace en la pestaña actual. Para abrir el enlace en una nueva ventana o pestaña, establece el target en _blank.

Para más información sobre la propiedad target, consulta esta página.

toService

Tiene cinco campos que se construyen para crear una URL como la siguiente: https://<rancher_server_url>/k8s/clusters/<CLUSTER>/k8s/namespace/<NAMESPACE>/service/<SCHEME>:<NAME>:<PORT>/proxy/<PATH>

Por ejemplo, un enlace a un servicio de monitorización se puede configurar de la siguiente manera:

  • nombre: rancher-monitoring-grafana

  • espacio de nombres: cattle-monitoring-system

  • vía: proxy/?orgId=1

  • puerto: "80"

  • esquema: http

Es necesario proporcionar ya sea la directiva toService o la directiva toURL.

toUrl

Puede ser cualquier enlace, incluso a enlaces fuera del clúster.

Es necesario proporcionar ya sea la directiva toService o la directiva toURL.

Ejemplos de enlaces

Ejemplo de enlace con toUrl

Este ejemplo de NavLink en YAML muestra un ejemplo de cómo configurar un NavLink a un panel de Grafana:

apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  name: grafana
spec:
  group: "Monitoring Dashboards"
  toURL: https://<RANCHER_SERVER_URL>/api/v1/namespaces/cattle-monitoring-system/services/http:rancher-monitoring-grafana:80/proxy/?orgId=1

Ejemplo de enlace con toService

Este ejemplo de YAML muestra un ejemplo de toService utilizado para el objetivo del enlace:

apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  annotations:
    key: annotation
    labels:
      key: label
  name: navlinkname
spec:
  description: This is a description field # Optional.
  group: "group1" # Optional. If not provided, the links appear standalone.
  iconSrc: data:image/jpeg;base64,[icon source string is clipped for brevity]
  label: This is a label # Optional.
  sideLabel: A side label. # Optional.
  target: _blank #Optional. _blank opens the link in a new tab or window.
  toService: # toService or #toUrl needs to be provided.
    name: rancher-monitoring-grafana
    namespace: cattle-monitoring-system
    path: proxy/?orgId=1
    port: "80"
    scheme: http
