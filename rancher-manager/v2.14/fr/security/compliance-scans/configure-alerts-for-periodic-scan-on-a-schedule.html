Configurer les alertes pour une analyse périodique planifiée

Il est possible d’exécuter un ClusterScan selon un horaire.

Une analyse programmée peut également spécifier si vous devez recevoir des alertes lorsque l’analyse est terminée.

Les alertes ne sont prises en charge que pour une analyse qui s’exécute selon un horaire.

L’application de conformité prend en charge deux types d’alertes :

  • Alerte à la fin de l’analyse : Cette alerte est envoyée lorsque l’exécution de l’analyse est terminée. L’alerte inclut des détails tels que le nom du ClusterScan et le nom du profil ClusterScanProfile.

  • Alerte en cas d’échec de l’analyse : Cette alerte est envoyée s’il y a des échecs de test dans l’exécution de l’analyse ou si l’analyse est dans un état Fail.

Conditions préalables

Avant d’activer les alertes pour rancher-compliance, assurez-vous d’installer l’application rancher-monitoring et de configurer les récepteurs et les itinéraires. Pour plus d’informations, consultez cette section.

Lors de la configuration des itinéraires pour les alertes rancher-compliance, vous pouvez spécifier la correspondance en utilisant la paire clé-valeur job: rancher-compliance-scan. An example route configuration is here.

Pour configurer des alertes pour une analyse qui s’exécute selon un horaire,

  1. Veuillez activer les alertes sur l’application rancher-compliance. Pour plus d’informations, consultez cette page.

  2. Dans le coin supérieur gauche, cliquez sur ☰ > Gestion des clusters.

  3. Sur la page Clusters, allez au cluster où vous souhaitez exécuter une analyse de conformité et cliquez sur Explorer.

  4. Cliquez sur conformité > Analyse.

  5. Cliquez sur Create.

  6. Choisissez un profil d’analyse de conformité. Le profil détermine quelle version du CIS Benchmark sera utilisée et quels tests seront effectués. Si vous choisissez le profil par défaut, alors l’opérateur de conformité choisira un profil applicable au type de cluster Kubernetes sur lequel il est installé.

  7. Choisissez l’option Exécuter l’analyse selon un horaire.

  8. Entrez une expression de cron valide dans le champ Horaire.

  9. Cochez les cases à côté des types d’alerte sous Alertes.

  10. Facultatives : Choisissez un Nombre de rapports conservés, qui indique le nombre de rapports maintenus pour cette analyse récurrente. Par défaut, ce nombre est de 3. Lorsque cette limite de conservation est atteinte, les rapports plus anciens seront supprimés.

  11. Cliquez sur Create.

Résultat : L’analyse s’exécute et se reprogramme selon l’horaire cron fourni. Des alertes sont envoyées lorsque l’analyse se termine si les itinéraires et le destinataire sont configurés sous rancher-monitoring l’application.

Un rapport est généré avec les résultats de l’analyse chaque fois que l’analyse s’exécute. Pour voir les derniers résultats, cliquez sur le nom de l’analyse qui apparaît.
