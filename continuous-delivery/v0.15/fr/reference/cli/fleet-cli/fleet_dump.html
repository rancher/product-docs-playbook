fleet dump

Déposer les données du cluster dans une archive.

Présentation

Déposer les données d’un cluster dans une archive, y compris :
* SUSE® Rancher Prime Continuous Delivery ressources gérées
* événements Kubernetes
* Métriques exposées par les contrôleurs Fleet

fleet dump <flags>

Drapeaux

  -p <path>         Path to the archive file to generate

Sortie

Table des matières

La commande dump produit une archive .tgz contenant les données suivantes :

  • Ressources brutes de Fleet, au format YAML :

    • Ensembles

    • BundleDeployments

    • BundleNamespaceMappings

    • Clusters

    • ClusterGroups

    • GitRepos

    • GitRepoRestrictions

    • HelmOps

  • Événements Kubernetes pour les espaces de noms suivants :

    • cattle-fleet-system

    • cattle-fleet-local-system

    • default

    • kube-system

    • chaque espace de noms contenant une ressource Fleet Cluster

  • Métriques exposées par les services monitoring-* vivant dans l’espace de noms cattle-fleet-system. Cela inclut généralement les services monitoring-gitjob et monitoring-fleet-controller, ainsi que leurs
    shardés homologues, le cas échéant.

Format

Les ressources brutes SUSE® Rancher Prime Continuous Delivery sont chacune stockées dans un fichier différent, nommé <kind>_<namespace>_<name>, où <kind> est l’un des noms de ressources brutes mentionnés dans la section précédente, en minuscules.

Les événements Kubernetes sont répartis dans des fichiers par espace de noms, chaque fichier étant nommé events_<namespace>. Ils contiennent un événement par ligne, chaque événement étant encodé en JSON.

Les métriques sont stockées dans des fichiers par service, chaque fichier étant nommé metrics_<service_name>. Leurs contenus sont GET corps de réponse reçus du point de terminaison /metrics de chaque service, tels quels.
